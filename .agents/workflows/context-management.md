# 컨텍스트 관리 워크플로우

장시간 개발 세션에서 컨텍스트가 비대해지는 것을 방지하고,
세션 전환 시 작업 연속성을 보장하기 위한 가이드입니다.

---

## 1. 세션 요약 스냅샷 (Manual Snapshot)

작업 마일스톤마다 현재 상태를 `Logging/` 폴더에 기록합니다.

### 스냅샷 저장 내용

- **완료된 작업**: PR 번호, 브랜치, Task 번호, 머지 상태
- **진행 중 작업**: 현재 브랜치, 미커밋 변경사항
- **다음 작업**: 우선순위에 따른 다음 Task
- **알려진 이슈**: 발견된 버그, 주의사항

### 스냅샷 타이밍

- PR 머지 후
- 3개 이상의 Task 완료 후
- 세션 종료 전 (필수)
- 에러 발생 시

## 2. 작업 상태 파일 유지

`todo/todo.md`를 항상 최신 상태로 유지합니다.

- 완료된 항목: `[x]` + PR 번호 표기
- 진행 중 항목: `[ ]` + (진행중) 표기
- 미시작 항목: `[ ]`

## 3. 세션 전환 시 복원

새 세션 시작 시 다음 파일을 우선 확인합니다:
// turbo

1. `todo/todo.md` - 전체 진행 상태 확인
   // turbo
2. `Logging/` 최신 날짜 파일 - 마지막 작업 내역 확인
   // turbo
3. `done/phase6_task.md` - Task 명세 확인
   // turbo
4. `git status && git branch && gh pr list` - Git/PR 상태 확인

## 4. 컨텍스트 절약 원칙

- 파일 전체 보기 대신 `view_file`에 StartLine/EndLine 지정
- 반복 확인 불필요한 파일은 재조회하지 않음
- 커밋 메시지에 충분한 정보를 포함하여 나중에 `git log`로 복원 가능
- 에러 발생 시 `errorCase/` 디렉토리에 기록하여 재발 방지

## 5. Hang 방지 (safe-terminal 연계)

- 모든 터미널 명령에 `WaitMsBeforeAsync` 적절히 설정
- `gh pr create`는 `--body-file` 사용 (인라인 body 대신)
- 대화형 명령 사용 금지 (`vi`, `nano`, `less` 등)

## 6. 임시파일 관리

- 임시파일은 **프로젝트 폴더 내** `.tmp/` 디렉토리에 생성
- 사용 후 즉시 삭제 (`rm -f .tmp/파일명`)
- `/tmp/` 에 임시파일 생성 금지
- `.tmp/` 는 `.gitignore`에 등록하여 커밋 방지

## 7. 작업 완료 후 컨텍스트 초기화

작업 블록 완료 시 다음을 수행:

1. Hang 프로세스 확인 및 종료
2. 임시파일 삭제
3. 현재 상태를 간결한 요약으로 압축
4. 불필요한 파일 내용을 재참조하지 않음
