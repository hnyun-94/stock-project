# 주식 리포트 자동화 시스템 상세 요구사항 정의서

## 1. 시스템 개요

본 시스템은 매일 3시간마다 주요 포털, 커뮤니티, 트렌드 데이터를 수집하고, AI를 활용해 핵심 내용을 요약한 뒤 사용자들에게 이메일과 카카오톡 봇을 통해 주식 관련 인사이트 리포트를 자동으로 발송하는 것을 목적으로 합니다.

## 2. 주요 기능 요구사항

### 2.1. 데이터 수집 (Data Ingestion)

다양한 소스에서 주식 관련 뉴스, 트렌드, 커뮤니티 화제성을 수집합니다.

- **언론사 뉴스**: 네이버 언론사 뉴스, 네이버 증권 뉴스, 다음 뉴스, 구글 뉴스
- **검색 트렌드**: 네이버 데이터랩, 구글 트렌드
- **커뮤니티 (화제성 및 여론)**: 네이버 증권 종토방, 디시인사이드 주식갤러리, 블라인드 주식게시판, 레딧(Reddit), 토스(Toss) 커뮤니티

### 2.2. 데이터 정제 및 요약 (Data Processing & AI Summarization)

- **데이터 클렌징**: 수집된 원본 데이터(HTML, JSON 등)에서 불필요한 광고, 태그, 중복 노이즈를 제거합니다.
- **AI 요약**: 정제된 데이터를 AI 언어 모델(OpenAI GPT, Claude 등)에 전달하여 핵심 이슈와 커뮤니티 반응을 3~4줄로 명확하게 요약합니다.
- **리포트 생성**: 요약된 내용을 바탕으로 가독성 좋은 리포트 템플릿(이메일용 HTML, 카카오톡 템플릿)으로 변환합니다.

### 2.3. 리포트 발송 (Notification Delivery)

- **발송 주기**: 3시간 간격으로 하루 8회 자동 발송 (예: 00시, 03시, 06시, 09시, 12시, 15시, 18시, 21시)
- **발송 채널**:
  1.  **이메일 (Email)**: 회원 가입된 이메일 주소로 상세 리포트 발송 (SMTP, AWS SES 등 이용)
  2.  **카카오톡 봇 (KakaoTalk Bot)**: 카카오톡 알림톡 또는 메시지 API를 통해 채널 추가된 사용자에게 알림 발송

### 2.4. 사용자 관리 (User Management)

- **관리 대상 정보**: 사용자 이름, 이메일 주소, 카카오톡 채널 ID/전화번호, 리포트 수신 여부 설정
- **데이터베이스**: 데이터를 쉽게 관리, 조회할 수 있는 환경 제공

## 3. 비기능 및 시스템 환경 요구사항 (호스팅 및 DB 대안)

서버 환경과 데이터베이스에 대한 무료 및 효율적인 구성 방안을 아래와 같이 제안합니다.

### 3.1. 사용자 데이터베이스 (DB) 추천

1.  **Notion API (노션 데이터베이스) - ⭐ 추천**
    - **장점**: 완전 무료, 별도의 관리자(Admin) 페이지를 만들 필요 없이 노션 앱에서 직관적으로 엑셀처럼 사용자 리스트를 추가/수정/삭제 가능합니다.
    - **단점**: 수천 명 단위로 넘어가면 API 속도가 느려질 수 있으나, 초기 시스템에 매우 적합합니다.
2.  **클라우드 무료 RDB (Supabase 또는 Neon DB)**
    - **장점**: 실제 서비스 레벨의 PostgreSQL 데이터베이스를 무료로 제공하며, 향후 확장성이 뛰어납니다.
    - **단점**: 별도의 데이터베이스 툴(DBeaver 등)을 다루거나 간단한 관리 뷰를 만들어야 합니다.

### 3.2. 서버 및 스케줄링 환경 추천

로컬 컴퓨터로도 구성할 수 있으나, 컴퓨터가 꺼지거나 절전모드에 들어가면 시스템이 멈춥니다. 따라서 **비용이 발생하지 않는 무료 호스팅(클라우드) 방안**을 추천합니다.

1.  **GitHub Actions (구독형 크론 스케줄러) - ⭐ 적극 추천**
    - **방식**: 서버를 24시간 띄워두는 것이 아니라, **3시간마다 한 번씩 깃허브 서버에서 코드를 실행**시키고 종료하는 방식입니다.
    - **장점**: 완전 무료이며, 서버 관리를 할 필요가 없고, 로컬 컴퓨터가 꺼져 있어도 알아서 동작합니다.
2.  **클라우드 프리 티어 가상 서버 (Oracle Cloud, AWS EC2)**
    - **방식**: 로컬 컴퓨터를 한 대 클라우드에 빌려 24시간 켜두는 방식입니다.
    - **장점**: 내 컴퓨터처럼 24시간 구동 프로그램을 돌리기 좋습니다. (Linux 환경)
    - **단점**: 직접 서버에 접속해 환경을 세팅(파이썬 설치 등)해야 하며, AWS의 경우 1년 후 과금될 수 있습니다. (Oracle은 평생 무료이나 가입이 까다로움)
3.  **내 로컬 컴퓨터 활용**
    - **방식**: 사용하고 계신 PC(또는 안 쓰는 남는 PC)에 스케줄러를 걸어두는 방식입니다.
    - **단점**: 1년 365일 컴퓨터와 인터넷을 켜두어야 합니다.

## 4. 제안하는 최적 시스템 아키텍처 아웃라인

1.  **언어 및 프레임워크**: Python (웹 크롤링 및 API 연동에 가장 강력하고 라이브러리가 많음)
2.  **데이터베이스**: Notion Database API (사용자 리스트 및 발송 히스토리 관리)
3.  **자동화 호스팅**: GitHub Actions (3시간마다 Python 스크립트 실행 트리거)
4.  **크롤링/스크래핑 라이브러리**: `requests`, `BeautifulSoup` (정적 사이트), `Selenium` 또는 `Playwright` (동적 렌더링 사이트 - 토스, 블라인드 등)
5.  **AI 요약 엔진**: OpenAI API (GPT-4o-mini 등 비용 효율적인 모델 사용)
6.  **메시지 전송**:
    - 이메일: Python 내장 SMTP + 구글 이메일(앱 비밀번호) 연동 또는 무료 발송 서비스(SendGrid 등)
    - 카카오톡: 카카오 디벨로퍼스 '메시지 API' 또는 '플러스친구 챗봇 API' 연동
