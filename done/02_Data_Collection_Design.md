# 데이터 수집 키워드 및 필터링 설계 문서

본 시스템은 시장의 전반적인 반응을 살피는 동시에, 사용자가 지정한 '관심 테마(키워드)'에 해당하는 뉴스나 반응을 우선적으로/비중 있게 수집하도록 설계됩니다.

---

## 1. 수집 파이프라인 아키텍처 (키워드 기반)

시스템의 크롤러는 다음과 같은 2가지 트랙으로 데이터를 수집합니다.

### 1.1. 트랙 A: 공통 시장 동향 (Market General)

- **목표**: 당일 주식 시장 전체의 분위기와 가장 핫한 종목 추적
- **동작 방식**:
  - 구글 트렌드 (한국)의 실시간/일별 급상승 검색어 중 경제/주식 관련 키워드 추출
  - 네이버 금융 주요 뉴스 (시황/전망 탭) 헤드라인 10~20개 무작위 수집
  - 디시인사이드 주식갤러리 일간 베스트 념글 (조회수/추천수 필터링) 제목 크롤링

### 1.2. 트랙 B: 관심 키워드 기반 (Keyword-Targeted)

- **목표**: 사용자가 특별히 알고자 하는 테마나 종목의 뉴스 비중을 높임
- **동작 방식 (Notion DB 연동)**:
  - Notion 사용자 데이터베이스에 `관심키워드(Tags)` 컬럼을 추가 (예: `["반도체", "인공지능", "제약", "엔비디아"]`)
  - Python 스크립트가 실행될 때, 수신 대상자들의 키워드 목록을 수집하여 중복을 제거한 **"통합 키워드 리스트"**를 만듦
  - 통합 키워드 리스트를 바탕으로 네이버 뉴스 검색 API 또는 네이버 파이낸스 뉴스 검색 크롤러를 통해 해당 키워드가 포함된 당일 최신/인기 기사를 N개씩 할당하여 추가 수집

---

## 2. 노션 데이터베이스 스키마 설계 반영

위의 '트랙 B'를 처리하기 위해 사용자 관리용 Notion 테이블은 다음과 같이 구성됩니다.

| 이름 (Title) | 이메일 (Email)  | 수신여부 (Checkbox) | 관심키워드 (Multi-select) |
| :----------- | :-------------- | :-----------------: | :------------------------ |
| 홍길동       | user@domain.com |         ✅          | `AI`, `반도체`, `테슬라`  |
| 김주식       | kim@domain.com  |         ✅          | `2차전지`, `엔비디아`     |
| ...          | ...             |         ...         | ...                       |

- **수집 모듈의 동작 로직**: 위 표의 데이터를 가져오면, 수집 모듈이 검색할 최종 키워드 셋은 `["AI", "반도체", "테슬라", "2차전지", "엔비디아"]` 가 됩니다.
- 이 키워드들을 기반으로 관련 뉴스 제목들을 수집하여 `[트랙 B 데이터]` 블록을 완성합니다.

---

## 3. Gemini 프롬프트 데이터 주입방식

결과적으로 AI 요약 모델(Gemini)에게 전달되는 데이터 스트링의 형태는 다음과 같이 구조화됩니다.

```text
=== 시장 일반 동향 데이터 (트랙 A) ===
[뉴스] 코스피 2700선 붕괴...
[트렌드] 에코프로비엠 거래량 급증...
[커뮤니티] 오늘 삼전 물린 개미들 상황...

=== 주요 관심 테마 및 집중 종목 데이터 (트랙 B) ===
[키워드: AI] ...
[키워드: 반도체] ...
[키워드: 테슬라] ...
```

이처럼 구조화된 텍스트를 AI에 전달하고 "시장 전체 동향 요약과 더불어, 위 관심 테마들(AI, 반도체 등)에 대한 이슈를 비중 있게 다뤄달라"는 프롬프트를 함께 전송하여 리포트를 생성합니다.
