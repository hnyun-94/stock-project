# 4.2 Daily 로깅 자동화 체계 (Task 4.2)

## 개요

- 매일매일 수동으로 작성하던 작업 내용을 자동화 시스템(크론/액션) 레벨에서도 투명하게 추적할 수 있도록 `logging` 패턴을 적용해야 함.

## 구현 방안

1. **Python 내장 `logging` 모듈 도입**
   - 콘솔 `print()` 문 단위로 출력되던 정보성/디버깅성 텍스트를 `logger.info()`, `logger.debug()`, `logger.error()` 형태로 통폐합 및 리팩토링.
2. **FileHandler 및 포매터(Formatter) 추가**
   - `logging` 폴더 하위에 날짜별(`2026-02-27.log` 등) 텍스트 파일로 자동 보존되는 `TimeRotatingFileHandler` 검토.
   - 출력 형태: `[2026-02-27 05:40:02][INFO][main] 시장지수 수집 시작` 등의 규격화된 포맷 개발.
3. **에러 폴더 분리 (ErrorCase 기록)**
   - `errorcase/` 경로에 치명적인 에러(Traceback)가 발생하면, 파일에 해당 에러의 전문과 템플릿(원인/시각 등)을 마크다운(`md`) 혹은 json 형태로 기록하여 영구 증빙용으로 저장하는 관리 체계 구축.

## 완료 조건

- 파이프라인 수행 후, `logging/YYYY-MM-DD.log` 파일에 파이프라인 진행 과정이 순차적으로 기록되어야 함.
- 임의 에러 발생 시 `errorcase/` 폴더 내에 정확히 에러 정보가 보존되며 로거에 에러 플래그가 기록되어야 함.
