# 오늘 바로 볼 숫자 비교표 고도화 계획서

## 요청 요약

- `오늘 바로 볼 숫자` 섹션에 전일/1주 비교와 등락 폭을 함께 보여주고 싶습니다.
- 같은 성격의 숫자는 소주제 단위로 묶어 표 형태로 더 읽기 쉽게 재구성해야 합니다.
- 추가 검토 아이디어가 있으면 함께 반영합니다.

## 역할별 검토

### PM

- 판단: 채택
- 근거 또는 이유: 현재 표는 "지금 값"만 보여줘서 왜 중요한지 연결은 되지만 변화의 맥락이 부족합니다.

### TPM

- 판단: 스키마 변경 없이 `report_snapshots` JSON을 확장해 비교 기준선을 누적
- 근거 또는 이유: 별도 테이블 추가 없이 기존 스냅샷 경로를 재사용하면 운영 영향이 작습니다.

### 기획자

- 판단: 숫자 표는 정량 항목 중심으로 정리하고, 테마 같은 정성 항목은 별도 판단표에 남기는 편이 더 명확
- 근거 또는 이유: `오늘 바로 볼 숫자` 섹션은 숫자·등락·비교 해석에 집중하는 것이 자연스럽습니다.

### 시니어 개발자

- 판단: crawler에서 당일 변화값을 채우고, builder에서 1D/1W 비교 행을 조립해야 함
- 근거 또는 이유: 과거 snapshot만으로는 전일 비교가 즉시 비지 않을 수 있어 현재 소스의 당일 change도 함께 활용해야 합니다.

### 개발자

- 판단: 변경 스트림은 세 갈래
- 근거 또는 이유:
  - 시장 지수 crawler change 파싱
  - scoreboard snapshot/비교 helper 추가
  - formatter/test 보강

### 품질 담당자

- 판단: payload와 렌더링 둘 다 회귀 테스트 필요
- 근거 또는 이유: 표 헤더, 비교 열, 그룹 행, fallback 문구가 동시에 바뀝니다.

### 신입개발자

- 판단: 비교 계산 helper와 표시 helper를 분리
- 근거 또는 이유: 지수/심리/검색 관심이 서로 다른 계산 규칙을 쓰므로 분리하는 편이 이해가 쉽습니다.

### 운영자

- 판단: snapshot JSON 확장은 허용 가능
- 근거 또는 이유: DB 스키마 변경 없이 현재 누적 경로를 유지합니다.

### 주식 전문가

- 판단: KOSPI/KOSDAQ뿐 아니라 환율·유가·금을 같은 숫자 비교표에 묶는 것이 해석 가치가 큼
- 근거 또는 이유: 지수 방향만 보면 놓치는 거시 압력을 함께 볼 수 있습니다.

### UI/UX 전문가

- 판단: `소주제 | 체크 대상 | 현재 값 | 전일 대비 | 1주 대비 | 읽는 포인트` 구조가 가장 스캔성이 좋음
- 근거 또는 이유: 같은 성격의 지표가 한 열에서 반복되면 그룹 읽기가 쉬워집니다.

### 퍼블리셔 전문가

- 판단: Markdown 단일 표 + 짧은 보조 메모가 이메일 렌더링에 가장 안전
- 근거 또는 이유: 다중 복잡 table이나 row span은 메일 클라이언트에서 불안정합니다.

## 검토 라운드

### 1차 검토

- 선택안 A: 기존 3열 표 유지 + 텍스트 설명만 보강
- 선택안 B: 비교 열과 그룹 열을 추가한 6열 표로 재구성
- 판단: B 채택
- 기각 이유:
  - A는 사용자가 요청한 "전일/1주 비교"를 충분히 충족하지 못합니다.

### 2차 검토

- 검토 내용: 비교 기준선 확보 방식
- 판단: 당일은 크롤러 change, 1주는 snapshot 누적값을 우선 사용
- 기각 이유:
  - 신규 이력 테이블 추가는 범위를 넘깁니다.

### 3차 검토

- 검토 내용: 추가 반영 아이디어
- 판단: 매크로 지표(미국 USD, WTI, 국제 금)와 비교 기준 설명 메모를 함께 반영
- 기각 이유:
  - 테마/보유 종목까지 같은 표에 섞으면 숫자 구간의 목적이 흐려집니다.

## 병렬 스트림

1. Stream A - market index change 파싱
- `src/crawlers/market_index.py`

2. Stream B - scoreboard 비교 로직
- `src/services/report_builder.py`

3. Stream C - formatter / 테스트
- `src/utils/report_formatter.py`
- `tests/services/test_report_builder.py`
- `tests/test_e2e_dryrun.py`

## 커밋 예산

1. crawler + scoreboard helper: 260 lines 이하
2. formatter + 테스트 + 계획/로그: 340 lines 이하

## 구현 범위

- 시장 지수/매크로 지표 change 파싱 보강
- scoreboard snapshot metric 저장
- `오늘 바로 볼 숫자`를 소주제 그룹형 비교 표로 재구성
- 전일 대비/1주 대비 표시 helper 추가
- 매크로 지표 행 추가
- 비교 기준 메모 추가

## 후속 범위

- snapshot 누적이 충분해지면 1개월 비교까지 확장
- 지수 시계열 전용 저장소 도입 시 sparkline 내장 표로 확장

## 수용 기준

- `오늘 바로 볼 숫자` 표에 전일 대비와 1주 대비 열이 생긴다.
- 시장 지수/매크로/심리 항목이 소주제 기준으로 묶여 보인다.
- KOSPI/KOSDAQ 외에 미국 USD, WTI, 국제 금이 함께 노출된다.
- 1주 기준선이 없을 때도 fallback 문구가 깨지지 않는다.
- 표준 품질 게이트를 통과한다.
