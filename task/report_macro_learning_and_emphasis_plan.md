# 리포트 경제 흐름/학습 카드/강조 품질 보강 계획

작성일: 2026-03-08  
상태: 완료

---

## 1) 배경

- 최근 리포트는 구조 자체는 개선됐지만, 사용자가 바로 읽고 판단하기엔 세 가지 빈틈이 남아 있었습니다.
- 첫째, 현재 경제 상황과 시장 트렌드를 한 번에 읽는 축이 약했습니다.
- 둘째, 리포트 하단에 초보 구독자가 배울 수 있는 짧은 학습 카드가 없어 구독 지속 가치가 낮았습니다.
- 셋째, 핵심 표현이 표와 본문 안에서 충분히 눈에 띄지 않아 스캔성이 떨어졌습니다.

---

## 2) 역할별 검토

### PM

- 리포트는 단순 요약이 아니라 “지금 무엇을 먼저 봐야 하는가”를 빠르게 알려줘야 합니다.
- 경제 상황/트렌드 축과 하단 학습 카드가 들어가면 재구독 가치와 체류 시간이 함께 올라갑니다.

### TPM

- 기존 payload 구조를 크게 흔들지 않고 `insight_lenses`, `learning_card` 중심으로 확장하는 것이 안전합니다.
- 강조 로직은 formatter 계층에서 처리해야 builder와 rendering 책임이 섞이지 않습니다.

### 기획자

- 상단은 판단, 하단은 학습으로 역할을 분리해야 콘텐츠 피로도가 낮습니다.
- 경제 흐름 섹션은 “경제 온도 / 자금 흐름 / 시장 화제”처럼 사용자가 바로 이해할 수 있는 별칭이 적합합니다.

### 시니어 개발자

- 학습 카드 용어 선택은 단순 문자열 포함 여부보다 현재 리포트의 주제어 우선순위를 반영해야 합니다.
- 강조 처리는 개별 문자열마다 수작업으로 넣기보다 공통 formatter 함수로 통일해야 유지보수가 쉽습니다.

### 개발자

- 기사 선별, 문장 교정, 강조 렌더링은 이미 존재하는 builder/formatter 확장으로 해결 가능합니다.
- 테스트는 payload 조립과 최종 Markdown 렌더링 양쪽에서 고정해야 회귀를 막을 수 있습니다.

### 품질 담당자

- `...`로 끊기는 요약, 붙여쓰기 오류, 표 안의 강조 누락은 모두 사용자 체감 품질 저하 요인입니다.
- 학습 카드가 무조건 일반 용어만 고르면 기대와 다른 결과가 나오므로 우선순위 규칙 검증이 필요합니다.

### 신입개발자

- 구조가 복잡해 보여도 builder는 데이터 선택, formatter는 표현 책임이라는 분리가 유지돼야 읽기 쉽습니다.
- 새 카드가 생기면 테스트 fixture에도 같은 구조를 넣어 줘야 전체 흐름이 이해됩니다.

### 운영자

- 리포트가 더 읽기 쉬워지면 피드백 노이즈가 줄고, “왜 이런 판단이 나왔는지 모르겠다”는 문의가 줄어듭니다.
- 하단 학습 카드가 있으면 초보 구독자의 이해도 편차를 운영이 따로 메우지 않아도 됩니다.

### 주식 전문가

- 경제 흐름은 뉴스 그 자체보다 환율, 금리, 수급, 테마 자금이 같은 방향을 가리키는지가 중요합니다.
- 학습 카드도 해당일 핵심 주제와 연결된 용어를 선택해야 실제 투자 판단 보조 자료가 됩니다.

### UI/UX 전문가

- 사용자는 표와 카드 안에서 핵심 단어를 먼저 스캔하므로 강조 체계가 필요합니다.
- 상단 판단 섹션과 하단 학습 섹션의 역할이 분리되면 정보 구조가 훨씬 명확해집니다.

### 퍼블리셔 전문가

- Markdown/이메일 환경에서는 색상보다도 굵기와 반복 가능한 텍스트 규칙이 더 안정적입니다.
- 표 셀 안의 핵심 키워드까지 공통 formatter로 강조해야 메일 클라이언트마다 일관되게 보입니다.

### 프롬프트 전문가

- 후처리 없는 모델 출력은 문장 완결성과 용어 통일성이 흔들리기 쉽습니다.
- builder에서 신호 문장 정제를 수행하고 formatter에서 표현을 통일하는 구조가 프롬프트 편차를 줄입니다.

### 컨텍스트 엔지니어

- 이번 작업은 AGENTS나 skill까지 건드릴 필요 없이 제품 출력 품질에 집중하는 것이 컨텍스트 효율상 맞습니다.
- 신규 규칙은 코드와 테스트, 계획 문서에만 닫아 두는 편이 토큰 비용 대비 효과가 높습니다.

## 3) 구현 판단

- 채택 1: 경제 흐름 섹션을 `경제 온도 / 자금 흐름 / 시장 화제` 구조로 유지하고, 상단 리포트의 핵심 해석 축으로 사용
- 채택 2: `learning_card`를 추가해 당일 리포트에서 가장 의미 있는 용어 1개를 `정의 + 오늘 왜 중요했는지 + 읽는 법`으로 제공
- 채택 3: 표와 카드 내부의 핵심 용어를 formatter 공통 함수로 강조
- 채택 4: 학습 카드 우선순위는 `AI -> HBM -> 환율 -> 금리 -> 수급` 순으로 잡아, 리포트 테마와 맞지 않는 일반 용어 선택을 줄임
- 기각 1: 별도 교육 섹션을 길게 추가하는 방식
  - 이유: 5~10분 리포트라는 제품 성격과 충돌하고 본문 집중도를 떨어뜨립니다.
- 기각 2: 색상만으로 중요도를 표현하는 방식
  - 이유: 메일 클라이언트별 렌더링 편차가 있어 안정성이 낮습니다.

## 4) 적용 범위

- `src/services/report_builder.py`

  - 문장 정제 보강

  - 표 셀 포함 공통 강조 렌더링

- `tests/test_e2e_dryrun.py`
- 기준선 문서
  - `done/completed_work_report.md`
  - `task/next_steps_roadmap.md`
  - `todo/todo.md`

## 5) 검증 기준

- 학습 카드가 현재 리포트 주제어를 우선 선택해야 합니다.
- 표 내부에서도 핵심 판단값이 강조되어야 합니다.
- 구조화 리포트 렌더링 테스트와 builder payload 테스트가 모두 통과해야 합니다.
- 전체 품질 게이트 통과 후 PR/merge까지 완료해야 합니다.
