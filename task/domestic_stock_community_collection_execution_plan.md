# 국내 주식 커뮤니티 P0 구현 계획서

## 요청 요약

- 국내 주요 주식 커뮤니티 검토 결과를 바탕으로 즉시 도입 가능한 수집/안전 정책을 구현합니다.
- 구현 범위는 `국내형 안전 필터 강화`, `증권플러스 인사이트 집계형 수집`, `블라인드 메타 보강`, `공통 파이프라인 연결`입니다.

## 역할별 검토

### PM

- 판단: P0는 커뮤니티 원문 확대보다 집계형/메타형 신호 추가가 맞습니다.
- 근거 또는 이유: 리포트 품질을 올리면서도 저신호 텍스트 유입을 최소화할 수 있습니다.

### TPM

- 판단: 브라우저 의존 소스는 기존 블라인드까지만 유지하고, 신규 소스는 공개 HTML 요청 기반으로 제한합니다.
- 근거 또는 이유: 운영 안정성과 실행 시간 증가를 완만하게 통제할 수 있습니다.

### 기획자

- 판단: 원문 제목보다 `투표 비율`, `참여 인원`, `핫이슈 제목` 같은 구조화 신호를 우선합니다.
- 근거 또는 이유: 사용자 입장에서 커뮤니티 감정은 "근거 축"이지 본문 자체가 아닙니다.

### 시니어 개발자

- 판단: 소스 추가보다 `source profile + safety rule + low weight merge` 구조를 먼저 넣어야 합니다.
- 근거 또는 이유: 앞으로 토스/팍스넷 같은 조건부 소스를 붙일 때 재사용성이 생깁니다.

### 개발자

- 판단: 변경은 네 갈래로 나눕니다.
- 근거 또는 이유:
  - `CommunityPost` 메타 확장
  - 안전 필터/소스 프로필 고도화
  - 증권플러스/블라인드 수집기 보강
  - main/tests 연결

### 품질 담당자

- 판단: 필터 테스트와 크롤러 파싱 테스트를 같이 보강해야 합니다.
- 근거 또는 이유: 국내형 금칙 패턴과 집계형 파싱은 회귀 포인트가 다릅니다.

### 신입개발자

- 판단: 소스별 허용 범위를 데이터 구조로 명시해야 합니다.
- 근거 또는 이유: 코드만 읽어도 어떤 소스가 제목 전용인지, 메타 사용 가능한지 보이게 해야 합니다.

### 운영자

- 판단: 기본 allowlist는 여전히 보수적으로 유지하되, 환경변수로 국내 소스를 명시 활성화하는 방식이 적절합니다.
- 근거 또는 이유: 운영 환경에서 단계적으로 켤 수 있어야 합니다.

### 주식 전문가

- 판단: 증권플러스는 집계형 여론, 네이버 종토방은 종목 심리, 블라인드는 직장인 체감으로 역할을 분리해야 합니다.
- 근거 또는 이유: 같은 커뮤니티 신호라도 의미 축이 다릅니다.

### UI/UX 전문가

- 판단: 커뮤니티 데이터는 기존 카드 구조를 유지하고, 제목 원문 노출량은 늘리지 않습니다.
- 근거 또는 이유: 원문이 많아지면 리포트 톤이 거칠어집니다.

### 퍼블리셔 전문가

- 판단: 메타는 텍스트 뒤 괄호 수준으로 붙이고, 과격 표현은 사전에 차단합니다.
- 근거 또는 이유: 이메일 렌더링 안정성과 톤 유지에 유리합니다.

## 검토 라운드

### 1차 검토

- 검토 내용: 즉시 후보 중 구현 난이도와 가치 비교
- 판단: `증권플러스 인사이트`, `블라인드 메타`, `국내형 안전 필터`를 우선 구현

### 2차 검토

- 검토 내용: 국내 커뮤니티 원문 사용 범위
- 판단: 본문/댓글/프로필 정보는 제외, 제목과 비식별 메타만 허용

### 3차 검토

- 검토 내용: 운영 반영 방식
- 판단: 기본 정책은 보수적으로 두고, 국내 소스는 `COMMUNITY_ENABLED_SOURCES`로 명시 활성화

## 선택안과 기각 이유

### 채택

- `CommunityPost`에 비식별 메타 필드 추가
- 국내형 비속어/민감정보 패턴 확장
- 소스별 허용 정책 정의
- 증권플러스 인사이트 집계 수집기 추가
- 블라인드에서 조회/좋아요 메타 수집

### 기각 또는 유보

- 토스 커뮤니티 원문 수집
- 디시 기본 활성화
- 댓글/본문/프로필/회사명 저장

### 기각 이유

- 개인정보·편향·노이즈 리스크가 아직 큽니다.
- 현재 리포트 구조에서 원문 커뮤니티 텍스트를 늘리는 것은 사용자 가치보다 부작용이 큽니다.

- 목표: 320 lines 이하

2. 수집기 + 파이프라인 + 테스트
- 목표: 380 lines 이하

## 구현 범위

- `CommunityPost`에 source/meta 요약이 가능한 필드 추가
- 국내형 수익 인증/평단/직장/정치 선동 패턴 필터 추가
- 소스별 허용 메타 정책 정의
- 증권플러스 인사이트 페이지에서 집계형 커뮤니티 신호 파싱
- 블라인드 수집기에서 조회/좋아요 메타 수집
- 메인 파이프라인에서 신규 소스를 안전 필터 후 병합

## 후속 범위

- 네이버 종토방 공통 수집 연동 여부 재검토
- usable rate 텔레메트리 추가
- 토스/팍스넷 조건부 실험군 정책 추가

## 수용 기준

- 국내형 저신호/민감정보 제목이 기존보다 더 많이 차단된다.
- 증권플러스 집계형 커뮤니티 신호를 공통 컨텍스트에 포함할 수 있다.
- 블라인드 메타가 보존되되, 회사명/본문은 수집하지 않는다.
- 기존 품질 게이트를 통과한다.

## 종합 판단

- 이번 P0는 국내 커뮤니티를 "많이 붙이는 작업"이 아니라, 안전하게 해석 가능한 공개 신호만 늘리는 작업입니다.
- 따라서 집계형 소스와 메타 기반 소스를 우선 도입하고, 원문 커뮤니티 확장은 후속으로 미룹니다.
