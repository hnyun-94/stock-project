# 리포트 근거 묶음 포맷터 개선 계획서

## 요청 요약

- 현재 리포트에서 `핵심 근거`, `지금 볼 것`, `관련 기사`가 같은 수준의 bullet로 길게 이어져 보여 가독성이 떨어집니다.
- 이 항목들을 하나의 묶음으로 재구성해 더 빨리 스캔할 수 있도록 개선합니다.

## 역할별 검토

### PM

- 판단: 채택
- 근거 또는 이유: 정보량을 줄이지 않고도 읽는 시간을 줄일 수 있는 개선입니다.

### TPM

- 판단: builder보다 formatter 중심 변경이 적절
- 근거 또는 이유: 현재 payload 구조는 이미 충분히 정보가 있으므로, 렌더링 계층만 정리하면 됩니다.

### 기획자

- 판단: `근거 묶음` 블록으로 묶고 항목 수는 유지하되 라벨 반복을 줄여야 함
- 근거 또는 이유: 사용자는 근거의 개수보다 "이 카드에서 무엇을 근거로 말하는지"를 한 덩어리로 보고 싶어 합니다.

### 시니어 개발자

- 판단: 공통 helper를 만들어 테마/보유/보조 지표/경제 흐름 섹션에 재사용
- 근거 또는 이유: 지금은 `_append_compact_brief`, `_append_lens_section`, `_append_card`가 유사 로직을 중복 구현합니다.

### 개발자

- 판단: formatter helper + 회귀 테스트 갱신으로 충분
- 근거 또는 이유: payload 스키마 변경 없이 구현 가능합니다.

### 품질 담당자

- 판단: Markdown과 HTML 둘 다 회귀 확인 필요
- 근거 또는 이유: 이메일 렌더링 결과가 핵심이므로 문구만 맞아도 충분하지 않습니다.

### 신입개발자

- 판단: `근거 묶음`은 4줄 이내로 정리하는 편이 이해가 쉽습니다.
- 근거 또는 이유: `핵심 근거`, `왜 중요한가`, `지금 볼 것`, `관련 기사` 4행이면 구조가 명확합니다.

### 운영자

- 판단: formatter 변경은 운영 리스크가 낮음
- 근거 또는 이유: 외부 수집이나 DB에 영향이 없습니다.

### 주식 전문가

- 판단: 근거 항목은 살아 있어야 하고, 단순 축약만 해서는 안 됨
- 근거 또는 이유: 읽기 편해져도 근거의 종류가 사라지면 해석력이 떨어집니다.

### UI/UX 전문가

- 판단: 라벨이 item마다 반복되는 구조보다, 묶음 안에서 한 줄 요약형으로 보이는 편이 낫습니다.
- 근거 또는 이유: 카드별 스캔 리듬이 더 일정해집니다.

### 퍼블리셔 전문가

- 판단: 표보다 짧은 리스트형 묶음이 메일 클라이언트 호환성에 더 안전
- 근거 또는 이유: 긴 셀과 다중 링크가 들어간 표는 모바일에서 깨질 수 있습니다.

- 기각 이유:

### 3차 검토

- 판단: `compact brief`, `lens`, `card` 계열 전체에 공통 적용

## 선택안과 기각 이유

### 채택

- 공통 `근거 묶음` helper 도입
- `핵심 근거`는 여러 개를 한 줄 또는 연속 줄로 묶고, `지금 볼 것`과 `관련 기사`는 각각 한 줄로 합침
- 기존 `세 가지 시각`, `다음 체크포인트` 구조는 유지

### 기각 또는 유보

- 근거 묶음을 별도 표로 렌더링
- payload 스키마 자체 변경

### 기각 이유

- 이번 요청의 핵심은 구조 개선이지 데이터 모델 변경이 아닙니다.
- 메일 환경에서는 짧은 bullet 묶음이 가장 안전합니다.

## 구현 범위

- 공통 `근거 묶음` 렌더링 helper 추가
- `핵심 근거`, `왜 중요한가`, `지금 볼 것`, `관련 기사`를 묶음으로 출력
- 테마/보유 종목/경제 흐름/보조 지표 섹션에 적용

## 후속 범위

- 필요 시 묶음 블록에 아이콘/배지 추가
- 샘플 카드별 항목 수 동적 축소 규칙 추가

## 수용 기준

- 동일 카드 안에서 근거 항목이 하나의 묶음처럼 읽힌다.
- 기존 payload 정보가 사라지지 않는다.
- E2E 테스트와 품질 게이트를 통과한다.

## 종합 판단

- 이번 문제는 데이터 부족이 아니라 렌더링 계층의 반복 표현 문제입니다.
- 따라서 formatter에서 공통 근거 묶음을 도입하는 것이 가장 비용 대비 효과가 큽니다.
