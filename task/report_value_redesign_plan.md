# 리포트 가치 중심 재설계 계획서

## 요청 요약

- 현재 리포트 구조를 재검토해 사용자 가치가 높은 내용과 낮은 내용을 분리한다.
- 최신 동향과 앞으로의 예측을 제공하되, 판단 근거가 보이는 구조로 재작성한다.
- 각 섹션은 짧은 요약, 근거가 되는 상세 설명, 긍정/중립/부정 시각, 다음 관찰 포인트를 포함해야 한다.
- 주식 초보도 이해할 수 있도록 쉬운 문장으로 쓰고, 하단에는 주요 용어 풀이를 제공해야 한다.
- 국장/미장 시작 30분 전과 마감 30분 이후에는 그 사이 이슈, 화제거리, 논쟁거리를 공통 요약으로 제공해야 한다.

## 필요/불필요 항목 분리

### 유지/강화가 필요한 항목

- 상단 헤드라인 변화
  - 이유: 직전 대비 바뀐 점을 가장 빠르게 파악할 수 있다.
- 보유 종목별 인사이트
  - 이유: 사용자가 실제로 행동 판단을 내릴 때 가장 직접적인 가치가 있다.
- 시간 흐름 기준 구간 요약(최근/오늘/1주/1개월)
  - 이유: 단기 노이즈와 중기 흐름을 분리해서 볼 수 있다.
- 최신 뉴스/검색 관심도/종목 뉴스 기반 근거
  - 이유: 판단의 출처가 보여야 리포트 신뢰가 생긴다.

### 축소/통합이 필요한 항목

- `AI` / `인공지능`처럼 사실상 같은 테마의 중복 섹션
  - 이유: 읽는 시간을 늘리지만 정보는 거의 늘어나지 않는다.
- `외부 커넥터 누적 데이터가 아직 부족합니다.` 반복 노출
  - 이유: 데이터 품질 경고 자체는 필요하지만 본문 여러 구간에 반복되면 가치가 낮다.
- `최근 7일 리포트 누적: N회` 같은 내부 운영성 지표
  - 이유: 사용자의 투자 판단에 직접적인 도움은 낮다.
- 실패 문자열 노출
  - 이유: 사용자에게 필요한 것은 장애 원인보다 대체 가능한 판단과 근거다.

### 제거 또는 하단 이관이 적절한 항목

- 내부 운영 문구 중심의 장황한 자동화 안내
  - 이유: 본문 가독성을 해친다.
- 반복되는 품질/부족 경고
  - 이유: 하나의 데이터 주의사항/신뢰도 노트로 충분하다.

## 역할별 검토

### PM

- 판단: 리포트는 "시장 설명"보다 "행동 판단 보조" 중심으로 바뀌어야 한다.
- 근거:
  - 현재 구조는 정보는 많아 보이지만, 사용자가 무엇을 먼저 봐야 하는지가 약하다.
  - 상단은 즉시 판단, 하단은 중장기와 설명 구조로 재편해야 한다.

### TPM

- 판단: report builder/formatter 중심 개편이 적절하다.
- 근거:
  - 데이터 수집 파이프라인을 크게 흔들지 않고도 payload 구조 개편으로 대부분 해결 가능하다.
  - 단, 세션 시간대 공통 요약과 중복 테마 병합을 위해 builder 입력은 약간 확장해야 한다.

### 기획자

- 판단: 모든 핵심 섹션은 동일한 읽기 패턴을 가져야 한다.
- 근거:
  - `짧은 요약 -> 근거 -> 긍정/중립/부정 -> 다음 체크포인트` 구조가 있어야 초보자도 읽기 쉽다.
  - UI/이메일 상에서도 섹션 간 기대치가 일정해진다.

### 개발자

- 판단: deterministic builder + AI 결과 압축의 혼합 방식이 적절하다.
- 근거:
  - 전부 AI 생성에 맡기면 품질 편차와 quota 의존성이 크다.
  - 이미 수집된 지표/뉴스/종목 뉴스만으로도 상당 부분은 규칙 기반으로 설명 가능하다.

### 신입개발자

- 판단: 일반화된 section card renderer가 필요하다.
- 근거:
  - 섹션마다 렌더링 규칙이 다르면 추후 유지보수가 어렵다.
  - 공통 카드 구조로 만들면 추가 섹션도 쉽게 붙일 수 있다.

### 운영자

- 판단: 세션 시간대 공통 요약은 조건부 노출로 제한해야 한다.
- 근거:
  - 항상 노출하면 중복 정보가 되고, 특정 시간대에만 가치가 높다.
  - 미국장 시간은 DST 영향을 받으므로 timezone-aware 계산이 필요하다.

## 검토 라운드

### 1차 검토

- 후보 A: 기존 구조 유지 + 문장만 다듬기
- 후보 B: payload/formatter 구조를 카드형으로 재설계

- 판단: B 채택
- 선택/기각 이유:
  - A는 중복, 근거 부재, 초보자 친화성 부족을 해결하지 못한다.
  - B는 한 번의 구조 변경으로 이후 확장성과 품질을 같이 확보한다.

### 2차 검토

- 검토 내용: AI 의존도 수준

- 판단: 핵심 구조는 deterministic, AI 결과는 보조 입력으로 사용
- 선택/기각 이유:
  - quota/모델 품질 이슈가 있어 AI 출력만 믿는 구조는 위험하다.
  - builder가 최종 구조와 설명 책임을 가져야 일관성이 생긴다.

### 3차 검토

- 검토 내용: 시간대 공통 요약 노출 방식

- 판단: 국장/미장 pre-open 30분, post-close 30분에서만 조건부 노출
- 선택/기각 이유:
  - 요구사항과 직접 일치한다.
  - 나머지 시간에는 상단을 불필요하게 길게 만들지 않는다.

### 4차 검토

- 검토 내용: 실제 출력 예시 기준의 정보 밀도와 행동 연결성

- 판단: 현재 카드 구조는 유지하되, 카드 안의 근거 설명을 더 구체화해야 한다.
- 선택/기각 이유:
  - 현 출력은 형식은 개선됐지만 "왜 중요한가"와 "그래서 무엇을 보면 되는가"가 아직 일반론에 머무는 구간이 있다.
  - 특히 테마/보유 종목 섹션은 뉴스 제목을 나열하는 수준을 넘어, 사용자 행동 판단에 직접 연결되는 의미 설명이 추가돼야 한다.

### 5차 검토

- 검토 내용: 장애/쿼터 이슈가 섹션 품질에 미치는 영향

- 판단: 운영 실패 문자열은 본문에서 제거하고, 데이터 기반 fallback 설명으로 치환해야 한다.
- 선택/기각 이유:
  - `RetryError`, `ClientError`, quota 초과 문구는 사용자 의사결정에 도움이 되지 않는다.
  - 실패 상황에서도 뉴스/지수/검색 흐름만으로 최소한의 구체적 설명이 유지돼야 리포트 가치가 지켜진다.

## 실제 출력 기준 추가 개선점

### 유지 판단

- 헤드라인 변화
  - 변화 감지가 빠르고 직전 리포트 대비 차이를 즉시 파악할 수 있다.
- 타임 윈도우 구조
  - 최근, 당일, 1주, 1개월의 읽기 순서가 자연스럽다.
- 보유 종목별 분리
  - 사용자가 실제로 보는 핵심 대상이므로 유지 가치가 높다.

### 보강 판단

- 테마/보유 종목의 "왜 중요한가" 설명
  - 현재는 구체 뉴스와 시장 영향의 연결이 약해 보일 수 있다.
- 긍정/중립/부정 시각의 근거성
  - 세 시각 모두 실제 기사/지수/수급 맥락과 연결되도록 구체화가 필요하다.
- 예측 문장의 확인 포인트
  - "앞으로 예상"은 예언처럼 보이면 안 되고, 어떤 숫자나 뉴스가 나오면 판단을 바꿀지 같이 제시해야 한다.

### 제거/차단 판단

- 운영 장애 문자열 직접 노출
  - 사용자 본문에서는 제거한다.
- 반복적인 데이터 부족 문구
  - 섹션마다 반복하지 않고, 신뢰도 메모 수준으로 압축한다.

## 종합 판단

- `report_builder`를 카드형 payload 생성기로 재설계한다.
- `report_formatter`는 카드형 공통 렌더러를 사용해 모든 섹션을 같은 읽기 패턴으로 출력한다.
- `main.py`는 theme/holding/news/community/time 정보를 builder에 더 전달한다.
- `AI`/`인공지능` 같은 중복 테마는 병합한다.
- glossary, session issue brief, data confidence note를 추가한다.
- 운영 실패 문구를 자동 제외하고, 뉴스/지수 기반 설명형 fallback을 builder 단계에서 강화한다.
- 각 카드는 `무슨 의미인지 -> 왜 중요한지 -> 그래서 무엇을 볼지`를 자연어로 풀어 주식 초보도 읽을 수 있게 만든다.

## 병렬 스트림

- 스트림 A: report builder 구조 개편
- 스트림 B: formatter 및 이메일 렌더링 호환 유지
- 스트림 C: session-time common brief / glossary / theme dedupe
- 스트림 D: 테스트 업데이트 및 회귀 방지

## 수용 기준

- 리포트 상단에서 `무엇이 바뀌었는지`, `지금 무엇을 볼지`, `왜 그런지`가 바로 보인다.
- 시장/테마/보유 종목 섹션이 모두 `요약/근거/긍정/중립/부정/다음 체크포인트` 구조를 가진다.
- 같은 의미의 중복 테마가 병합된다.
- 국장/미장 시작 30분 전과 마감 30분 이후에 공통 이슈 요약이 노출된다.
- 하단에 용어 풀이가 자동으로 붙는다.
- 관련 테스트와 전체 품질 게이트가 통과한다.
