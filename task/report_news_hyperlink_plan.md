# 리포트 주요 뉴스 하이퍼링크 개선 계획서

## 요청 요약

- 주요 뉴스와 시황 카드에서 바로 원문 기사를 열 수 있도록 하이퍼링크를 제공합니다.
- 시장/테마/보유 종목 카드가 plain text 제목만 보여주는 문제를 개선합니다.

## 역할별 검토

### PM

- 판단: 링크 제공은 리포트 신뢰도를 높이는 직접적인 사용자 가치 개선입니다.
- 근거 또는 이유: 사용자는 요약만 읽는 것보다 근거 기사 원문을 바로 확인할 수 있어야 안심하고 판단할 수 있습니다.

### TPM

- 판단: 링크는 formatter가 아니라 payload 단계에서 구조화해야 합니다.
- 근거 또는 이유: 시장/테마/종목 카드마다 동일한 렌더링 패턴으로 재사용하려면 builder가 링크 데이터를 보존해야 합니다.

### 기획자

- 판단: 링크는 상단 시장 판단과 하단 종목 카드 모두에 자연스럽게 붙어야 합니다.
- 근거 또는 이유: 상단은 시장 전체 근거 확인용, 하단은 실제 보유 종목 검증용으로 읽는 목적이 다릅니다.

### 시니어 개발자

- 판단: `related_links` 같은 공통 필드를 카드 schema에 추가하는 쪽이 맞습니다.
- 근거 또는 이유: 세부 섹션마다 문자열 안에 링크를 직접 섞는 방식은 구조가 깨지기 쉽습니다.

### 개발자

- 판단: builder에는 관련 기사 추출 helper, formatter에는 링크 렌더링 helper를 추가하면 충분합니다.
- 근거 또는 이유: 현재 Markdown->HTML 경로가 이미 링크를 지원하므로 데이터만 넣어주면 됩니다.

### 품질 담당자

- 판단: payload에 URL이 들어오는지와 최종 Markdown/HTML에 hyperlink가 살아있는지 둘 다 검증해야 합니다.
- 근거 또는 이유: builder와 formatter 중 한쪽만 테스트하면 실제 이메일 출력 회귀를 놓칠 수 있습니다.

### 신입개발자

- 판단: 링크는 `label/url` 구조로 고정하는 편이 읽기 쉽습니다.
- 근거 또는 이유: 이후 채널이 늘어도 같은 필드를 재사용할 수 있습니다.

### 운영자

- 판단: 이번 변경은 외부 호출 증가 없이 기존 수집 기사 링크를 재사용하므로 운영 부담이 거의 없습니다.
- 근거 또는 이유: 추가 크롤링이나 API 호출 없이 이미 가진 `NewsArticle.link`를 노출하는 수준입니다.

### 주식 전문가

- 판단: 시장 판단과 종목 판단이 어떤 기사에 근거했는지 바로 보여주는 것이 좋습니다.
- 근거 또는 이유: 사용자는 요약과 원문을 함께 봐야 과도한 단정이나 오독을 줄일 수 있습니다.

### UI/UX 전문가

- 판단: 링크는 `관련 기사` 블록으로 분리해 시선 흐름을 해치지 않게 배치하는 편이 좋습니다.
- 근거 또는 이유: 본문 문장 중간에 URL을 섞는 방식은 읽기 흐름을 깨뜨립니다.

### 퍼블리셔 전문가

- 판단: Markdown 링크를 formatter에서 그대로 HTML `<a>`로 변환하는 현재 구조가 가장 안전합니다.
- 근거 또는 이유: 메일 클라이언트 호환성과 유지보수성을 함께 만족합니다.

## 종합 판단

- card payload에 `related_links`를 추가하고, formatter가 `관련 기사` 섹션으로 렌더링하도록 구현합니다.
- 시장 요약, 시간대 카드, 관심 테마, 보유 종목 카드에 우선 적용하고 테스트로 고정합니다.
