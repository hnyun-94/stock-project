## 2) 우선순위 백로그

### P0. 운영 알림 자동화

목표:
- 외부 소스 실패/지연을 실시간 감지하고 운영 채널에 자동 알림

구현 항목:
- `external_connector_runs` 기반 source별 1h/24h 실패율 계산
- 임계치 초과 시 텔레그램 관리자 알림 전송
- 알림 중복 방지(쿨다운)

완료 기준:
- 장애 시 5분 내 운영 알림 1회 이상 발생
- 정상 구간에서는 알림 스팸 없음

---

### P1. 텔레메트리 리포트화

목표:
- 사용자/운영자가 데이터를 신뢰할 수 있도록 품질 지표를 정기 리포트로 노출

구현 항목:
- source별 성공률/평균 지연/최근 오류 사유 요약
- 일별 집계 배치(최근 7일)
- 리포트 본문 또는 운영 문서에 자동 삽입

완료 기준:
- 최근 7일 지표가 리포트에 재현 가능
- source별 품질 저하 구간을 수치로 확인 가능

---

### P2. 도메인 지표 고도화

목표:
- OpenDART/FRED/SEC 지표를 단순 count에서 해석 가능한 시계열 신호로 확장

구현 항목:
- OpenDART 카테고리별 1D/7D 변화율
- FRED 시리즈 다중화(금리/물가/실업 등) + 레짐 신호
- SEC 지표(샘플링) 변화량 기반 이상치 탐지

완료 기준:
- 최소 3개 이상 신규 해석 지표가 리포트에 반영
- 회귀 테스트 추가 및 품질 게이트 통과

---

### P3. 문서 자동 동기화

목표:
- 코드/문서 상태 불일치를 줄여 운영 혼선을 방지

구현 항목:
- `todo`, `phase6_task`, `logging` 동기화 점검 스크립트
- CI에서 문서 불일치 경고(또는 실패) 규칙 도입

완료 기준:
- 문서 상태 불일치 시 PR 단계에서 탐지
- 수동 문서 정리 빈도 감소
