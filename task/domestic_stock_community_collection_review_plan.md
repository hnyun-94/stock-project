# 국내 주식 커뮤니티 수집 확장 검토 계획서

## 요청 요약

- 토스 커뮤니티를 포함한 국내 주요 주식 커뮤니티가 현재 어떤 공개 표면을 가지는지 확인합니다.
- 그중 실제로 유의미한 데이터를 수집할 수 있는 사이트를 분류하고, 수집 진행 가능 여부를 검토합니다.
- 비속어, 혐오 표현, 민감정보, 과도한 개인 투자 인증처럼 편향적이거나 쓸모 없는 데이터가 많은지도 함께 판단합니다.

## 현재 기준선

- 현재 레포에는 네이버 종토방, 디시인사이드 주식 갤러리, Reddit WSB 수집기와 블라인드 동적 크롤러가 존재합니다.
- 다만 커뮤니티 안전 정책은 기본 allowlist를 매우 보수적으로 유지하고 있어, 현재 기본 허용 소스는 `reddit_wallstreetbets` 뿐입니다.
- 안전 필터도 아직은 제목 중심 필터라서 국내 커뮤니티를 넓히려면 소스별 위험 정책을 더 세분화해야 합니다.

## 역할별 검토

### PM

- 판단: 커뮤니티 확장은 가치가 높지만 "더 많이"보다 "더 믿을 만하게"가 우선입니다.
- 근거 또는 이유: 국내 주식 커뮤니티는 참여량이 많아 체감 민심을 잡기 좋지만, 노이즈와 선동이 매우 높아 잘못 넣으면 리포트 신뢰도를 떨어뜨립니다.

### TPM

- 판단: 소스는 `공개 HTML`, `공개 CSR`, `앱/인증 기반`, `보호 우회 필요` 네 종류로 나눠야 합니다.
- 근거 또는 이유: 수집 가능성은 데이터 가치보다 접근 방식과 운영 리스크에 더 크게 좌우됩니다.

### 기획자

- 판단: 원문 게시글 수집과 집계형/큐레이션형 신호 수집을 분리해야 합니다.
- 근거 또는 이유: 동일한 "커뮤니티"라도 어떤 곳은 원문 제목이 유용하고, 어떤 곳은 설문 비율이나 편집 요약만 가져오는 편이 낫습니다.

### 시니어 개발자

- 판단: 국내 커뮤니티 확장은 크롤러보다 정책 계층이 먼저입니다.
- 근거 또는 이유: 현재 필터는 욕설/연락처/주민번호 수준만 막고 있어, 직장명, 수익 인증, 평단 공개, 정치 선동 같은 국내형 노이즈를 충분히 다루지 못합니다.

### 개발자

- 판단: 이번 라운드에서 현실적인 분류는 세 단계입니다.
- 근거 또는 이유:
  - 즉시 후보: 공개 페이지 + 제목/메타만으로도 가치가 있는 소스
  - 조건부 후보: 데이터 가치는 높지만 앱/CSR/민감정보 이슈가 큰 소스
  - 보류 후보: 보호 우회가 필요하거나 저신호가 너무 높은 소스

### 품질 담당자

- 판단: 소스별로 "수집 성공"보다 "쓸 만한 데이터 비율"을 측정해야 합니다.
- 근거 또는 이유: 국내 커뮤니티는 수집은 쉬워도 실제 usable title 비율이 낮은 경우가 많습니다.

### 신입개발자

- 판단: `source risk profile`을 별도 맵으로 두는 편이 이해하기 쉽습니다.
- 근거 또는 이유: 허용 여부, title/body 사용 여부, 이미지 허용 여부, 필터 강도, 출력 가중치를 소스별로 다르게 가져가야 합니다.

### 운영자

- 판단: 공개성, robots, 인증 요구, anti-bot 리스크를 같이 봐야 합니다.
- 근거 또는 이유: 한 번 수집이 되는 것과 운영 경로에서 안정적으로 유지되는 것은 전혀 다른 문제입니다.

### 주식 전문가

- 판단: 종목별 군중심리는 네이버 종토방, 투자자 투표/집계는 증권플러스, 직장인 투자 체감은 블라인드가 상대적으로 유효합니다.
- 근거 또는 이유: 각 커뮤니티의 사용자군이 달라 신호의 성격도 다릅니다.

### UI/UX 전문가

- 판단: 원문 커뮤니티 문장을 그대로 보여주기보다, 집계와 짧은 요약으로 변환해야 합니다.
- 근거 또는 이유: 그대로 인용하면 저품질 문구가 리포트 전체 톤을 무너뜨립니다.

### 퍼블리셔 전문가

- 판단: 커뮤니티 신호는 `여론 요약`, `상위 논점`, `투표 비율` 형태가 가장 안전합니다.
- 근거 또는 이유: 제목 원문은 짧게 참고할 수 있지만, 과격한 표현이 그대로 드러나는 구조는 메일 리포트에 맞지 않습니다.

## 검토 라운드

### 1차 검토

- 검토 내용: 국내 주요 커뮤니티의 공개 표면과 접근 방식 조사
- 판단: 네이버 종토방, 블라인드, 디시, 팍스넷, 토스증권, 증권플러스는 의미 있는 검토 대상입니다.

### 2차 검토

- 검토 내용: 데이터 유의미성 대비 수집 난이도와 민감정보 위험 비교
- 판단: `네이버 종토방`, `증권플러스 인사이트형 집계`, `블라인드 제목+메타`가 우선 후보입니다.

### 3차 검토

- 검토 내용: 현재 레포의 필터/리포트 구조에 맞는 도입 범위
- 판단: 토스 원문 커뮤니티와 개인 투자 인증형 데이터는 보류하고, 집계형 신호 중심으로 들어가는 것이 맞습니다.

## 외부 커뮤니티별 관찰

### 1. 토스증권 커뮤니티

- 관찰:
  - 토스증권 웹 표면에는 `인기 급상승 커뮤니티`, `지금 뜨는 카테고리`가 노출됩니다.
  - 커뮤니티 약관에는 특정 커뮤니티회원의 매매경향, 팔로우 관계, 내투자정보 기반 프로필/활동 알림이 명시되어 있습니다.
  - 동시에 약관은 개인신용정보와 실시간 매매현황은 제외한다고 명시합니다.
- 해석:
  - 데이터 가치 자체는 매우 높습니다.
  - 다만 개인 투자 성향, 팔로우 관계, 보유/관심 종목 변동과 맞물리는 구조라 개인정보/행동데이터 민감도가 높습니다.
  - 공개 웹에서는 커뮤니티 존재는 드러나지만, 원문 게시글을 안정적으로 SSR 형태로 수집할 표면은 확인되지 않았습니다.
- 판단: `조건부 후보`
- 수집 원칙:
  - 원문 게시글/프로필/수익 인증/팔로우 그래프는 수집하지 않음
  - 공개 웹에서 안정적으로 얻을 수 있는 카테고리/랭킹/집계형 메타만 허용 검토
  - 공식 API 또는 명시적 공개 표면이 없으면 보류

### 2. 네이버 Npay 증권 종목토론실

- 관찰:
  - 종목토론 게시판이 공개 HTML로 제공됩니다.
  - `주주오픈톡`은 자산 연결 기반 주주 인증이 필요하므로 별도 채팅 영역입니다.
  - 페이지는 `클린봇으로 착한게시글만 모아보세요`와 시세조종/허위사실 유포 주의를 함께 노출합니다.
  - `robots.txt`는 `/item/board.naver?code=*`는 허용하고, pagination 수집은 비허용합니다.
- 해석:
  - 종목별 군중심리를 가장 직접적으로 반영하는 국내 공개 소스입니다.
  - 동시에 선동, 허위정보, 정치·감정 글도 많아 품질 편차가 큽니다.
  - 첫 페이지 제목만 수집하는 방식은 기술적으로 무난하고 robots에도 상대적으로 맞습니다.
- 판단: `즉시 후보`
- 수집 원칙:
  - 제목과 링크만 수집
  - `주주오픈톡` 제외
  - page 순회 금지, 첫 페이지 상위 글만 사용
  - 종목별 1~3건 + 강한 필터 + 낮은 출력 가중치

### 3. 블라인드 주식·투자

- 관찰:
  - 공개 topic 페이지에서 `조회수`, `좋아요`, `댓글` 수와 작성시간이 함께 보입니다.
  - 게시글 옆에 회사명 레이블이 붙고, 투자 고충/수익 경험/감정적 표현이 그대로 드러납니다.
- 해석:
  - 직장인 투자자 심리와 체감 온도를 읽기에는 유효합니다.
  - 반면 회사명, 직군, 투자 경험, 대출/손실 서사 등 개인 맥락이 강해 민감도가 높습니다.
  - 현재 레포에도 Playwright 기반 수집기가 이미 있어 기술적 진입비용은 높지 않습니다.
- 판단: `즉시 후보(제한적)`
- 수집 원칙:
  - 제목 + 조회/좋아요/댓글 메타만 사용
  - 본문, 댓글, 회사명 라벨은 저장/노출 금지
  - 개인 서사성 제목은 점수 하향 또는 제거

### 4. 디시인사이드 주식 갤러리

- 관찰:
  - 추천글 목록은 공개 HTML로 접근됩니다.
  - 실제 추천글 제목에는 비속어, 혐오성 표현, 정치성 문구가 섞여 있습니다.
  - 사이트 자체에 차단 단어/닉네임/IP 설정 기능이 따로 있을 정도로 노이즈 전제가 강합니다.
- 해석:
  - 밈성 민심, 과열/공포 분위기, 화제 키워드를 잡는 데는 쓸 수 있습니다.
  - 그러나 리포트용 근거 데이터로 쓰기에는 비속어·혐오·정치 편향 비율이 매우 높습니다.
- 판단: `조건부 후보`
- 수집 원칙:
  - 추천글 제목만 수집
  - source weight는 매우 낮게 유지
  - 욕설/정치/젠더/선정성 필터를 현행보다 크게 강화
  - 필터 통과율이 낮으면 기본 비활성화

### 5. 팍스넷 커뮤니티

- 관찰:
  - 커뮤니티 허브가 공개되어 있고 `시황분석`, `데이트레이딩`, `자유게시판`, `깔깔유머방`이 분리돼 있습니다.
  - 실제 노출 글은 시황 외 잡담, 자기매매일지, 비유적 표현, 저신호 텍스트가 많이 보입니다.
- 해석:
  - 역사적으로 투자자 커뮤니티로 의미는 있으나, 현재 공개면 기준으로는 저신호 텍스트 비율이 높습니다.
  - `시황분석` 같은 특정 board만 한정하면 일부 활용 여지는 있습니다.
- 판단: `보류에 가까운 조건부 후보`
- 수집 원칙:
  - 자유/유머 게시판은 제외
  - 시황분석만 별도 실험 대상으로 검토
  - title quality score 기준 미달 시 소스 제외

### 6. 증권플러스 커뮤니티/인사이트

- 관찰:
  - 인사이트 페이지는 `커뮤니티 투표` 참여 인원과 결과 비율을 공개합니다.
  - `개미의 핫글`류 콘텐츠는 커뮤니티 토론방의 대표 글과 반응을 편집 요약해 보여줍니다.
  - 동시에 `매수가격 공개`, `수익률 인증`, `수익금 인증` 같은 개인 투자 인증형 콘텐츠도 노출됩니다.
- 해석:
  - 원문 커뮤니티보다 편집/집계된 공개 신호는 매우 유의미합니다.
  - 특히 설문 비율, 참여 인원, 핫이슈 요약은 리포트에 넣기 좋은 구조입니다.
  - 다만 개인 수익률 인증형 콘텐츠는 리포트 근거로 쓰면 과도하게 선정적이고 편향됩니다.
- 판단: `즉시 후보(집계형 전용)`
- 수집 원칙:
  - 인사이트의 설문 결과, 집계 비율, 편집형 요약만 사용
  - 수익 인증, 평단 공개, 보유 수량 공개형 글은 제외
  - 원문 댓글/이미지/개인 인증 요소는 수집하지 않음

### 7. 에펨코리아 주식 게시판

- 관찰:
  - 검색 및 직접 접근 안정성이 낮고, 기존 검토 이력상 보호 우회 이슈가 예상됩니다.
  - 일반 커뮤니티 특성상 주식 외 잡담과 슬랭 비중이 높을 가능성이 큽니다.
- 해석:
  - 이번 라운드 기준으로는 수집 안정성과 정보 밀도 모두 우선순위가 낮습니다.
- 판단: `보류`

## 수집 후보 분류

### A. 즉시 후보

1. 네이버 Npay 증권 종목토론실
- 수집 형태: 제목 + 링크
- 기대 신호: 종목별 국내 개인투자자 심리
- 위험: 선동, 허위정보, 정치성 제목

2. 증권플러스 인사이트 기반 집계형 커뮤니티 신호
- 수집 형태: 투표 결과, 참여 인원, 편집형 핫이슈 요약
- 기대 신호: 구조화된 군중 심리와 비교 프레임
- 위험: 개인 수익 인증형 콘텐츠 혼입

3. 블라인드 주식·투자
- 수집 형태: 제목 + 조회/좋아요/댓글 수
- 기대 신호: 직장인 투자자 체감과 관심사
- 위험: 회사명/직군/개인 재무 서사 노출

### B. 조건부 후보

1. 토스증권 커뮤니티
- 수집 형태: 공개 집계 메타가 확인될 때만 제한적 사용
- 기대 신호: 실제 투자자 행동 기반 인기 흐름
- 위험: 내투자정보, 팔로우 관계, 활동 알림 기반 개인정보 민감성

2. 디시인사이드 주식 갤러리
- 수집 형태: 추천글 제목만
- 기대 신호: 과열/공포 밈성 민심
- 위험: 비속어, 혐오, 정치 편향, 저신호 비율 높음

3. 팍스넷 시황분석
- 수집 형태: 제목만 제한적 실험
- 기대 신호: 올드 리테일 투자자 시황 문맥
- 위험: 잡담과 자기일지 비율, 정보 품질 불균일

### C. 보류

1. 에펨코리아 주식 게시판
- 이유: 접근 안정성 및 anti-bot 리스크

2. 네이버 주주오픈톡
- 이유: 자산 연결 기반 인증 채팅으로 공개 웹 수집 대상이 아님

3. 토스/증권플러스의 개인 프로필·수익 인증·보유 내역형 데이터
- 이유: 개인정보/과도한 개인 투자 인증 리스크

## 편향적이거나 쓸모 없는 데이터 기준

### 제거 대상

- 욕설, 혐오 표현, 젠더/지역/정치 선동 문구
- 연락처, 이메일, 주민등록번호, 계좌/실시간 매매 정황
- 회사명, 직군, 개인 식별 닉네임 등 직장/개인 추적성이 높은 메타
- `몰빵`, `풀매수`, `영끌`, `빚투`, `평단 공개`, `수익률 인증`, `수익금 인증` 중심 제목
- 단순 조롱, 감탄사, 밈 짤 설명처럼 해석 가치가 낮은 제목

### 제한 사용 대상

- 과열/공포 심리를 보여주는 짧은 제목
- 커뮤니티 투표 비율과 참여 인원
- 편집자가 요약한 핫이슈 정리
- 조회/좋아요/댓글 같은 비식별 메타

## 선택안과 기각 이유

### 채택

- 네이버 종토방 제목 수집
- 증권플러스 인사이트형 집계 데이터 수집
- 블라인드 제목+메타 수집
- 토스/디시/팍스넷은 조건부 실험군으로 분리

### 기각 또는 유보

- 토스 원문 커뮤니티 직접 수집
- 수익 인증/평단 공개/보유 수량 공개형 글 수집
- 댓글 본문, 이미지, 닉네임, 회사명, 프로필 기반 수집
- 에펨코리아 같은 보호 우회 필요 소스

### 기각 이유

- 현재 필터 수준으로는 국내 커뮤니티 특유의 민감도와 노이즈를 충분히 감당하기 어렵습니다.
- 개인 투자 인증형 데이터는 리포트의 근거보다 선정성과 편향을 더 키울 가능성이 큽니다.
- 보호 우회성 수집은 운영 안정성과 정책 리스크가 큽니다.

## 병렬 스트림

1. Stream A - 소스 정책 정립
- source risk profile
- 허용 범위와 출력 가중치 정의

2. Stream B - 안전 필터 확장
- 국내형 비속어/정치/민감정보 패턴 확장
- 직장명/프로필/수익 인증 제거 규칙 추가

3. Stream C - 신규 수집기 도입
- 네이버 종토방 정교화
- 블라인드 메타 수집
- 증권플러스 인사이트 집계 수집

## 커밋 예산

1. 정책/필터 계층
- 300 lines 이하

2. 증권플러스 인사이트 + 블라인드 메타
- 350 lines 이하

3. 네이버 종토방 정교화 + 회귀 테스트
- 350 lines 이하

## 구현 범위

- 이번 라운드는 검토와 분류가 범위입니다.
- 구현이 시작되면 `즉시 후보`만 먼저 도입합니다.
- 원문 커뮤니티보다 집계형 신호를 우선 적용합니다.

## 후속 범위

- 국내형 커뮤니티 안전 필터 사전 고도화
- community usable rate 측정용 telemetry 추가
- 소스별 weight와 fallback 전략 도입

## 수용 기준

- 국내 주요 주식 커뮤니티가 공개성, 유의미성, 민감정보 위험, 운영 가능성 기준으로 분류된다.
- 즉시 수집 후보와 조건부/보류 후보가 구분된다.
- 비속어/민감정보/저신호 데이터에 대한 배제 원칙이 정의된다.

## 종합 판단

- 현재 기준에서 가장 실용적인 국내 커뮤니티 확장 방향은 `네이버 종토방 + 증권플러스 집계형 + 블라인드 제목 메타` 조합입니다.
- 토스 커뮤니티는 데이터 가치가 높지만 개인 투자 맥락과 결합되는 구조라, 공개 집계 메타 외에는 당장 수집 대상으로 삼지 않는 것이 맞습니다.
- 디시와 팍스넷은 기술적으로 수집 가능하더라도 저신호·편향 위험이 커서 저가중치 실험군 이상으로 다루면 안 됩니다.
