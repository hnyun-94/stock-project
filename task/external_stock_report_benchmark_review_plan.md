# 외부 주식 리포트 벤치마크 검토 계획서

## 요청 요약

- 인터넷에 있는 다양한 주식 리포트를 참고해 현재 리포트에 적용할 수 있는 요소를 검토합니다.
- 단순 사례 나열이 아니라, 현재 레포트 구조에 실제로 붙일 수 있는 항목과 데이터 확장 필요 항목을 구분합니다.

## 현재 리포트 구조 기준선

- 현재 리포트는 `지금 결론`, `오늘 바로 볼 숫자`, `경제 상황과 트렌드`, `시간대 압축판`, `관심 테마`, `보유 종목별 인사이트`, `보조 지표 해석`, `데이터 신뢰도`, `경제 상식`, `용어 풀이` 구조입니다.
- 렌더링 경로는 이메일용 Markdown/HTML이므로, 대형 차트팩·복잡한 PDF 레이아웃보다 짧은 표/카드/배지형 표현이 맞습니다.

## 역할별 검토

### PM

- 판단: 외부 벤치마크는 가치가 높음
- 근거 또는 이유: 현재 리포트는 핵심은 잡혔지만, "이번 주 무엇을 보나"와 "왜 이 숫자가 중요한가"를 더 선명하게 만들 여지가 있습니다.

### TPM

- 판단: 형식 벤치마크와 데이터 벤치마크를 분리해야 함
- 근거 또는 이유: 외부 리포트의 모든 요소가 현재 수집 범위에서 즉시 구현 가능한 것은 아닙니다.

### 기획자

- 판단: 참고 포맷은 세 층으로 나뉨
- 근거 또는 이유:
  - 원페이지 요약형
  - 주간/월간 전략형
  - 섹터/스타일 레이팅형

### 시니어 개발자

- 판단: 현재 구조에는 "캘린더", "레이팅", "다기간 퍼포먼스", "시나리오/전술 분리"가 특히 잘 맞음
- 근거 또는 이유: 기존 payload/formatter 구조를 크게 깨지 않고 추가할 수 있습니다.

### 개발자

- 판단: 적용 가능 항목은 세 단계로 나눠야 함
- 근거 또는 이유:
  - 즉시 적용 가능: 현재 데이터와 payload만으로 구현 가능
  - 데이터 확장 후 가능: 새 수집 또는 snapshot 누적이 더 필요
  - 보류: 이메일 환경과 맞지 않거나 과도하게 무거움

### 품질 담당자

- 판단: 외부 포맷 모방 시 과밀화 리스크를 경계
- 근거 또는 이유: 정보가 많아질수록 현재 리포트의 5~10분 스캔성이 깨질 수 있습니다.

### 신입개발자

- 판단: 리포트 타입별로 좋은 패턴을 따로 이름 붙여 관리하는 편이 낫다
- 근거 또는 이유: `주간 캘린더형`, `섹터 레이팅형`, `데이터 센터형`처럼 패턴이 분리되면 구현 우선순위를 잡기 쉽습니다.

### 운영자

- 판단: 공개 웹 리포트에서 참고할 것은 주로 "정보 구조"와 "정량 축"이어야 함
- 근거 또는 이유: 원문 전체를 따라가면 지속적인 유지보수 비용이 커집니다.

### 주식 전문가

- 판단: 현재 리포트에 가장 부족한 것은 "상대 비교 축"과 "섹터/스타일 관점"임
- 근거 또는 이유: 개별 뉴스 해석은 늘었지만, 시장 내 상대 강약을 보는 프레임은 아직 얇습니다.

### UI/UX 전문가

- 판단: 외부 리포트의 장점은 대부분 시각화보다 위계 설계에 있음
- 근거 또는 이유: 무엇을 먼저 보고, 무엇을 다음에 보는지가 분명합니다.

### 퍼블리셔 전문가

- 판단: PDF형 차트팩보다 표와 미니 카드 레벨로 번역해야 함
- 근거 또는 이유: 이메일 클라이언트에서 긴 도표나 복잡 레이아웃은 쉽게 깨집니다.

## 검토 라운드

### 1차 검토

- 검토 내용: 외부 리포트 유형 분류
- 판단: `주간 퍼포먼스형`, `전략/전술형`, `섹터 레이팅형`, `원페이지 데이터 센터형`, `관전 포인트형`으로 나눠 검토

### 2차 검토

- 검토 내용: 현재 리포트와 구조 매핑
- 판단: 현재 구조에는 `관전 포인트`, `주간 캘린더`, `상대 퍼포먼스 표`, `레이팅 매트릭스`, `전술/전략 분리`가 가장 잘 맞음

### 3차 검토

- 검토 내용: 구현 현실성
- 판단: 이번 라운드에서는 적용 가능 항목만 우선 제안하고, 새 데이터 소스가 필요한 항목은 후속 범위로 분리

## 적용 우선순위

### 즉시 적용 가능

1. `이번 주 관전 포인트 3개`
- 이유: 현재 뉴스/테마/체크포인트 데이터를 재가공하면 바로 가능

2. `이번 주 체크 일정`
- 이유: 현재는 일정 캘린더가 없어 행동 연결이 약함
- 필요 데이터: 실적 일정, 경제지표 일정 소스 추가 또는 간단 수집

3. `관심 테마/보유 종목 레이팅`
- 이유: Schwab식 `확대/중립/경계`는 현재 카드 구조에 바로 얹기 쉬움

4. `호재 / 리스크` 이중 구조
- 이유: 현재 `긍정/부정`은 있지만, 실무적으로는 두 줄 요약이 더 빠르게 읽힘

5. `깊게 보기` 카드 1개
- 이유: Fidelity식 `A closer look`을 현재 `경제 상황과 트렌드`에 적용 가능

### 데이터 확장 후 적용

1. `주간 퍼포먼스 표`
- 필요 데이터: S&P500/Nasdaq/섹터/원달러/WTI/Gold의 1W/1M/YTD 시계열

2. `데이터 센터`
- 필요 데이터: 밸류에이션, 집중도, 이익성장률, 변동성 등 비교 수치

3. `섹터 비교표`
- 필요 데이터: 미국/한국 섹터별 상대 수익률 및 레이팅 기준선

4. `전술 vs 전략` 자산군 view
- 필요 데이터: 기간별 관점과 전략 레이블을 분리할 별도 판단 규칙

### 보류 또는 비권장

1. 다페이지 PDF 차트팩 복제
- 이유: 이메일 환경과 맞지 않음

2. 과도한 미국 중심 advisor jargon
- 이유: 현재 한국 사용자 중심 리포트와 톤이 어긋남

3. 대형 이미지/복잡 SVG 차트
- 이유: 메일 클라이언트 호환성이 떨어짐

## 선택안과 기각 이유

### 채택

- 현재 리포트 구조 안에 `관전 포인트`, `주간 일정`, `레이팅`, `호재/리스크`, `깊게 보기`, `주간 퍼포먼스 표`를 단계적으로 이식
- 외부 리포트의 강점은 형식이 아니라 정보 위계와 비교 축으로 번역
- 즉시 적용 가능 항목과 데이터 확장 필요 항목을 분리해 P0/P1/P2로 추진

### 기각 또는 유보

- 다페이지 PDF 차트팩 복제
- 이미지 중심 차트 의존 구조
- 미국 자문사용 용어를 그대로 가져오는 문체
- 데이터 확장 없이 `데이터 센터`, `섹터 매트릭스`, `전술/전략`을 성급히 도입하는 방식

### 기각 이유

- 현재 리포트는 이메일 기반이라 복잡한 시각화와 긴 차트팩이 맞지 않습니다.
- 현재 수집 범위로는 일부 패턴을 흉내만 내게 되어 신뢰도가 떨어질 수 있습니다.
- 사용자는 한국 시장 중심의 실용적 요약을 기대하므로, 미국 기관투자자 문체를 그대로 가져오면 어색합니다.

- 현재 리포트는 `개인화`와 `설명력`은 빠르게 개선됐지만, 외부 우수 사례 대비 `상대 비교`, `일정 연결`, `레이팅`, `원페이지 데이터 센터` 감각이 아직 약합니다.
- 가장 효과적인 다음 단계는 새 섹션을 많이 늘리는 것이 아니라, `관전 포인트`, `주간 일정`, `레이팅`, `퍼포먼스 표` 같은 벤치마크 요소를 현재 구조 안에 보수적으로 이식하는 것입니다.
- 특히 Fidelity, Schwab, J.P. Morgan, LPL의 패턴은 현재 레포트에 현실적으로 옮기기 쉽고, BlackRock/Vanguard의 구조적 위계는 섹션 역할 분리에 직접 도움이 됩니다.
