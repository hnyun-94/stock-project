---
description: 에러 발생 로깅 가이드
---

작업 중 오류(Error)가 발생하면 바로 종료하거나 무시하지 말고 다음 단계를 정확히 따르세요.

1. `errorcase` 폴더 내에 오늘 날짜와 에러 유형을 이름으로 하는 파일(`YYYY-MM-DD_에러명.md`)을 생성합니다.
2. 해당 파일에 다음 포맷으로 에러 발생 내용을 세밀하게 추가합니다.

   ```markdown
   # 발생 시간: [HH:MM]

   ## 1. 에러 상황 (Symptom)

   - (어떤 작업을 하려다 에러가 발생했는지, 터미널 로그나 Traceback 핵심 요약)

   ## 2. 발생 원인 분석 (Cause)

   - (에러가 왜 발생했는지에 대한 분석 내용)

   ## 3. 조치 내용 (Resolution)

   - (수정한 코드 내역이나 환경설정 변경 내용 기록)
   ```

3. 에러 로깅 후, 해당 조치를 반영하여 작업을 재개합니다.
